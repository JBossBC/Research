# Code standard

##  前端引用的页面需要精简，不要引用重

引用重:如果浏览器没有缓存的话可能会下载两次。
引用的没有用到:加大浏览器的加载带宽。

## 代码逻辑必须要方便迭代

避免用下标去指代数据，应该在保证环境变化的情况下依然能够指代该数据。

## 同步和异步一定要考虑清楚，能尽量异步一定要异步，不然会影响客户体验

## 注意代码复用问题

##  修复缺陷的时候可以将原来代码注释掉方便对比

##  不能根据前端的代码传参给后端，如果代码复用了会导致后端代码的逻辑性不够严谨，后端设计到参数类型转换的话，必须要进行一次参数验证，能不能转
